# Product

## Register

product

## Users

Agencies and resellers managing WhatsApp AI automation for multiple clients. Power users who are comfortable with tools like Make.com or n8n — they're not technical developers but they move fast, expect software to behave predictably, and judge quality instantly. They're often running this on behalf of paying clients, so the tool's credibility reflects directly on them.

## Product Purpose

InboxAI is a self-hosted WhatsApp inbox with AI auto-replies. It lets users connect a WhatsApp number via QR, manage all inbound conversations in one place, configure keyword auto-replies and multi-step flows, and integrate with external automation tools via webhooks and API. Success means a user can manage their entire WhatsApp business communication workflow without touching WhatsApp directly.

## Brand Personality

Sharp, reliable, quietly confident. Three words: **precise, modern, trustworthy**. This is professional software for professionals — not a toy, not a generic SaaS template, not a WhatsApp clone. It should feel like the kind of tool an agency would proudly demo to a client.

## Anti-references

- Raw developer-tool aesthetic (unstyled inputs, plain HTML tables, no visual hierarchy)
- Flashy SaaS with gradient text, glassmorphism cards, hero metric layouts
- WhatsApp green copycat — should have its own visual identity
- Heavy enterprise UI with overwhelming chrome and dense gray dashboards

## Design Principles

1. **Earn familiarity** — use standard affordances (nav, forms, buttons) executed with precision. No invented controls.
2. **Density on demand** — chat lists and message views can be dense; settings panels stay spacious and readable.
3. **State is signal** — every interactive element must have a clear hover/focus/active/disabled state. Nothing feels dead.
4. **The sidebar disappears** — navigation should not compete with the conversation. Dark sidebar, light content, clean split.
5. **Professional confidence** — the visual language should make an agency proud to show this to a client.

## Accessibility & Inclusion

WCAG AA minimum. All interactive elements keyboard-navigable. Focus rings visible. Sufficient color contrast for body text (≥4.5:1). Reduced motion respected.
